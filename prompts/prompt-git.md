# Versionamento de código

- Crie commits seguindos o padrão Conventional Commits(feat, fix, docs, style, refactor, perf, test, build, ci, chore)
- Utilize as tasks desenvolvidas e bugs resolvidos e separe em vários commits segregados
- Sempre faça commit após cada task concluída
- Não faça commit de arquivos não relacionados ao projeto
- UTILIZE SEMPRE Inglês nos commits
- Quando houver necessidade de fazer mais de um commit, faça um commit para cada task concluída SEMPRE SEPARE OS COMMITS
- Não faça push das alterações, apenas faça o commit
- Quando houver arquivos sensíveis, remova-os antes de fazer o commit e insira no .gitignore
